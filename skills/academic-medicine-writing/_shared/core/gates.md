# Medicine Gates

Stop and ask before drafting or strengthening any claim when these are unresolved:

- population or clinical setting
- endpoint/outcome definition
- comparator or control
- sample size or cohort provenance
- ethics / IRB / consent status
- trial registration or protocol status when applicable
- adverse events or safety evidence for safety claims
- source anchor for diagnostic, treatment, or deployment claims

## Input Sufficiency Check

Run this before loading a long workflow or drafting long-form prose.

Input sufficiency: enough / partial / insufficient.

- `enough`: the route is clear and the available evidence, draft, or artifact is sufficient to act
  without inventing medical facts.
- `partial`: the route is clear but some facts constrain the output; proceed only within the safe
  boundary, list blockers, and avoid expanding the task.
- `insufficient`: the route, evidence anchor, manuscript identity, or requested artifact is unclear;
  ask for the smallest missing input before long-form work.

Use the same check for all six intent classes: `full-draft-from-evidence`, `package-completion`,
`draft-revision`, `figure-table-only`, `citation-only`, and `submission-review`. For `partial` or
`insufficient` inputs, report severity and correction priority instead of burying the issue in
process notes.

## Blocking Gates

Block "submission-ready" wording when any applies:

- the study type/checklist is unknown
- a required checklist item is `missing` without a reason
- ethics, consent, data access, or trial registration statements are invented or unresolved
- a clinical claim lacks population/outcome/comparator/timeframe anchoring
- diagnostic, treatment, safety, or deployment language exceeds the evidence boundary
- completed checklist, flow diagram, protocol, SAP, or supplement is required but not planned
