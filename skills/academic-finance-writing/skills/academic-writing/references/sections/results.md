# Finance Results

## Result Hierarchy Gate

Before drafting results, rank findings:

1. central result that answers the paper question;
2. mechanism or channel evidence;
3. heterogeneity that changes interpretation;
4. robustness or falsification evidence;
5. secondary descriptive facts.

The first results subsection should carry the central result unless the confirmed Paper Framework
chooses another order for a stated reason.

## Section Movement

For each result unit:

1. start with the economic conclusion;
2. cite the table, figure, proposition, or model output;
3. state coefficient, alpha, elasticity, return spread, default probability, valuation effect,
   comparative static, or prediction error;
4. translate magnitude into a finance benchmark such as sample mean, standard deviation, basis
   points, market capitalization, assets, Sharpe ratio, default rate, or policy cost;
5. state uncertainty and inference;
6. explain what belief changes and what threat remains.

Do not write a table tour. Do not begin with "Table 3 reports" unless the next clause states the
economic result.

## Null And Weak Results

Report null or weak results when they affect the argument. Explain whether they rule out a mechanism,
bound an effect, expose low power, or show the limitation of the design.
