# Finance Paper-Type Profile Boundary

Paper-type profiles define the default section structure for a finance manuscript type. A profile
owns section structure and section priority only: section list, order, naming, count, structural
role, priority contract, compression order, and minimum protection for the sections that carry the
paper's main contribution.

Profiles do not own finance domain, identification strategy, econometric method, page limit, venue
format, citation policy, figure design, data-code policy, or final submission checks. Route those
concerns to domain/method references, venue/template/version-target cards, citation, figure, data,
and review files.

## Paper Framework Hard Default

At the Paper Framework stage, the selected profile is a hard default, not loose inspiration. By
default, reproduce the profile's section table exactly, including section names and order, and quote
it as the canonical list in the Paper Framework's "Structure vs paper-type profile" comparison.

Deviate only when the actual contribution, evidence, venue requirement, or explicit user request
genuinely cannot fit the selected structure. Every split, merge, rename, addition, or reorder must
be surfaced and approved at the Paper Framework checkpoint. Silent structural deviation is a
workflow violation.

## Length Boundary

Paper-type profiles must not invent absolute page or word limits. Absolute length comes from the
selected venue card, official template instructions, selected version-target card, or current
official source. Profiles may mark sections as primary core, evidence core, support, or
compress-first so the Paper Framework can allocate the active venue/version budget.
