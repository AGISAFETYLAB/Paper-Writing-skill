# Paper-Type Profile Boundary

Paper-type profiles define the default section structure for a manuscript type. A paper-type profile
owns section structure and budget only: section list, order, naming, count, page or word budget,
priority contract, compression order, and minimum floors for the sections that carry the paper's
main evidence.

Profiles do not own writing style, reviewer strategy, citation policy, claim admissibility,
terminology, venue rules, figure design, or final submission checks. Route those concerns to the
shared core, venue, citation, figure, and review files.

## Paper Framework Hard Default

At the Paper Framework stage, the selected profile is a hard default, not loose inspiration. By
default, reproduce the profile's section table exactly, including the section names and order, and
quote it as the canonical list in the Paper Framework's "Structure vs paper-type profile"
comparison.

Deviate only when the actual contribution, evidence, venue requirement, or explicit user request
genuinely cannot fit the selected structure. Every split, merge, rename, addition, or reorder must
be surfaced and approved at the Paper Framework checkpoint. Silent structural deviation is a
workflow violation.
